# Palantir Gotham — Inspiration Notes

Gotham is a military/intel data fusion platform. Personal finance is the same problem: siloed data, hidden patterns, decisions under uncertainty.

## What Gotham Does

- **Data fusion** — structured + unstructured sources into one unified object graph
- **Network/link analysis** — relationships between entities (people, places, events)
- **Geospatial overlay** — real-time situational awareness on maps
- **Pattern detection / anomaly surfacing** — AI finds what humans miss
- **Attribute-level access control** — permissions per data point, not per table
- **Decision support (mission planning)** — not just dashboards, actionable next steps

## Epiphany Equivalents

| Gotham | Epiphany |
|--------|----------|
| Data fusion across siloed sources | Wealthsimple + SnapTrade + manual accounts → one model |
| Entity graph | Security → holdings → transactions → news events |
| Anomaly surfacing | "Dining spend 3x last month", unusual portfolio swings |
| Geospatial overlay | Portfolio exposure by geography (flights feature) |
| Object-level access control | Multi-user household with per-account visibility |
| Mission planning / decision support | Autopilot — goal-based actions, not just charts |

## Highest-Value Steal

**Object graph model.** Every entity (stock, account, transaction) has typed relationships. Query the graph, not flat tables.

Current state: accounts → holdings (shallow).  
Target: transactions → categories → merchants → recurring patterns → insights.

This unlocks "find the connection" analysis — the core of what makes Gotham powerful.
